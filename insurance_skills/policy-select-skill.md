# Policy Select Skill

## Skill 目标

本 Skill 用于在用户提问涉及具体保险责任、理赔判断、药品报销、医院限制、赔付比例、免赔额、材料要求时，先确认用户所指的具体保险产品或产品版本，避免把不同产品、不同版本、不同人群规则混用。

本 Skill 不直接回答理赔结论，只负责判断是否已经具备足够的产品上下文。

---

## 适用场景

当用户问题涉及以下任一内容时，必须先执行本 Skill：

- 能不能赔
- 能不能报销
- 能不能理赔
- 这个药能不能赔
- 住院自费能不能赔
- 特药能不能赔
- 质子重离子能不能赔
- CAR-T 能不能赔
- 免赔额是多少
- 赔付比例是多少
- 需要什么材料
- 医院范围
- 药房范围
- 既往症
- 等待期
- 保障责任
- 保额
- 保险责任
- 除外责任
- 免责条款

---

## 产品识别原则

### 1. 用户已明确产品版本

如果用户明确说出产品或版本，例如：

- 2025 版沪惠保普通版
- 沪惠保普通版
- 关爱版
- 沪惠保关爱版
- 新市民版
- 沪惠保新市民版

则直接使用对应产品知识库，不要再要求用户重复选择。

---

### 2. 用户只说“沪惠保”

如果用户只说“沪惠保”，但没有说明是普通版、关爱版还是新市民版，则不得直接套用普通版规则。

应判断用户问题是否依赖版本差异。

如果问题涉及以下内容，必须提示用户选择版本：

- 住院自费医疗费用保险金
- 医保结算前置条件
- 医疗互助帮困计划
- 当地医保结算
- 既往症定义
- 投保范围
- 人群资格
- 赔付比例差异
- 理赔前置条件

提示方式：

“这个问题需要先确认你买的是哪一版沪惠保：普通版、关爱版，还是新市民版。三版在投保人群、住院理赔前置条件和既往症定义上不完全一样。”

---

### 3. 用户没有提供任何保单信息

如果用户问的是具体理赔问题，但没有提供产品名称，应先提示用户选择或上传保单。

提示方式：

“这个问题需要结合具体保单判断。你可以先选择保险产品，或者上传保单/产品说明书，我再帮你判断是否能赔。”

---

### 4. 健康档案中已有保单

如果系统提供的用户上下文中只有一个有效保单，可以默认使用该保单，但回答中应说明：

“我先按你当前档案中的 XX 产品来判断。”

如果系统提供的用户上下文中有多个保单，应让用户选择。

---

## 沪惠保 2025 三版识别规则

当前测试中存在三个产品版本：

### 普通版

识别关键词：

- 普通版
- 2025 版沪惠保
- 上海基本医保
- 上海市基本医疗保险
- 上海医保卡

核心人群：

参加上海市基本医疗保险的在保人员。

住院责任关键前置条件：

特定住院自费医疗费用需经上海基本医疗保险结算。

---

### 关爱版

识别关键词：

- 关爱版
- 社区医疗互助帮困计划
- 医疗互助帮困计划
- 互助帮困
- 帮困计划

核心人群：

参加上海市市民社区医疗互助帮困计划的人员。

住院责任关键前置条件：

需先获得“上海市市民社区医疗互助帮困计划”医疗费用补助。

---

### 新市民版

识别关键词：

- 新市民版
- 新市民
- 外卖
- 快递
- 物流
- 大型企业务工人员
- 当地医保
- 非上海医保
- 未经当地医保结算

核心人群：

在上海部分大型企业工作，并参加当地基本医疗保险的务工人员。

住院责任关键前置条件：

原则上需经当地基本医疗保险结算。

但如果未经当地基本医疗保险结算，新市民版仍可按降低后的比例赔付：

- 非既往症人群：20%
- 既往症人群：10%

---

## 版本差异保护规则

当用户没有明确版本时，不得直接使用普通版作为默认版本。

尤其以下问题必须先确认版本：

1. “住院自费能不能赔？”
2. “没有走医保能不能赔？”
3. “既往症怎么定义？”
4. “我是不是符合投保范围？”
5. “赔付比例是多少？”
6. “理赔前要不要先走医保？”
7. “互助帮困计划能不能赔？”
8. “我是外卖/快递/物流人员，能不能买/能不能赔？”

---

## 输出要求

如果产品版本已经明确，输出结构化路由结果：

```json
{
  "policy_status": "confirmed",
  "policy_name": "2025版沪惠保",
  "policy_version": "普通版/关爱版/新市民版",
  "next_skill": "claim-judge-skill"
}
```

如果需要用户选择版本，输出：

```json
{
  "policy_status": "need_select",
  "reason": "用户只提到沪惠保，但未说明具体版本；该问题涉及版本差异。",
  "candidates": [
    {
      "name": "2025版沪惠保普通版",
      "description": "适用于参加上海市基本医疗保险的在保人员"
    },
    {
      "name": "2025版沪惠保关爱版",
      "description": "适用于参加上海市市民社区医疗互助帮困计划的人员"
    },
    {
      "name": "2025版沪惠保新市民版",
      "description": "适用于在上海部分大型企业工作并参加当地基本医保的务工人员"
    }
  ]
}
```

---

## 禁止行为

1. 用户只说“沪惠保”时，禁止默认当成普通版。
2. 禁止把普通版住院医保结算规则套用到关爱版或新市民版。
3. 禁止把关爱版的互助帮困计划规则套用到普通版。
4. 禁止忽略新市民版“未经当地医保结算可按低比例赔付”的特殊规则。
5. 禁止在产品版本不明确时输出确定理赔结论。
